NexusHub MCP Tools

Filesystem Tools

fs_list_files

Lists files and directories in a specified path with optional filtering.

fs_read_file

Reads and returns the content of a file from the filesystem.

fs_write_file

Writes content to a file, creating it if it doesn't exist.

Database Tools

db_execute_query

Executes a read-only SQL query and returns the results.

db_list_tables

Lists all tables available in the database.

db_describe_table

Returns the schema information for a specified table.

db_insert_data

Inserts data into a specified database table.

Docker Tools

docker_list_containers

Lists all Docker containers with their status information.

docker_start_container

Starts a stopped Docker container by name or ID.

docker_stop_container

Stops a running Docker container by name or ID.

docker_get_container_logs

Retrieves the logs from a Docker container with options for filtering.

Vector & Search Tools

ingest_docs

Ingests documents into the vector database for semantic search.

vector_search

Performs semantic search over ingested documents.

serper_search

Performs web search via the Serper API with options for filtering.

All of these tools are available directly through the NexusHub MCP server. Use the Help tab for more detailed information about each tool's parameters and usage.

Help & Documentation

Welcome to NexusHub!

NexusHub is a centralized MCP (Model Context Protocol) server that provides various tools and capabilities to Claude Code and other AI assistants.

Quick Start Guide

  1. Configure API Keys - Go to the API Keys tab and set up the credentials required for each service.
  2. Check Service Status - Ensure all services are running (green status indicators).
  3. Add Documents - Add documents to the vector database for context-aware assistance.
  4. Use Prompts - Explore the prompts tab for examples of how to effectively instruct the AI.

System Requirements

  • Docker and Docker Compose installed
  • Port 8001 available for the HTTP server
  • Internet connection for API services

Available Tools

NexusHub provides the following tools to Claude Code:

Filesystem Tools

  • fs_list_files - Lists files and directories
  • fs_read_file - Reads file content
  • fs_write_file - Writes content to a file

Database Tools

  • db_execute_query - Executes a read-only SQL query
  • db_list_tables - Lists database tables
  • db_describe_table - Describes table schema
  • db_insert_data - Inserts data into a table

Docker Tools

  • docker_list_containers - Lists Docker containers
  • docker_start_container - Starts a Docker container
  • docker_stop_container - Stops a Docker container
  • docker_get_container_logs - Retrieves container logs

Search Tools

  • serper_search - Performs web search via Serper API

Vector Tools

  • ingest_docs - Ingests documents into the vector store
  • vector_search - Performs similarity search over documents

Other Services

  • Memory MCP - Provides knowledge graph capabilities
  • GitHub MCP - Enables GitHub repository interactions
  • Brave Search MCP - Provides web search capabilities

Frequently Asked Questions

How do I restart a service?

You can restart individual services by clicking the "Restart" button next to each service card on the dashboard.

Are my API keys stored securely?

Yes, all API keys are stored in the server's database and are not exposed in logs or client-side code.

Can I use custom models for vector embeddings?

Yes, you can configure custom embedding models by modifying the server configuration.

How do I add more tools?

You can extend the NexusHub server by implementing additional tool definitions in the server code.

Can I run NexusHub without Docker?

Yes, but Docker is recommended for the best experience. Check the documentation for running without Docker.

Troubleshooting

Service Connection Issues

If a service is showing as offline:

  1. Check if the Docker container is running
  2. Verify that API keys are correctly configured
  3. Look at the container logs for error messages
  4. Try restarting the service

API Key Problems

If API services are not working:

  1. Ensure the API key is valid and not expired
  2. Check for any usage limits or restrictions
  3. Try refreshing the environment variables

Vector Database Issues

If document ingestion fails:

  1. Check file formats and sizes
  2. Ensure the vector database is properly initialized
  3. Verify disk space and permissions

Getting Support

For additional help:

  • Check the server logs for detailed error messages
  • Visit the GitHub repository for documentation
  • Open an issue for bug reports or feature requests
